# Sprint 1 Plan (2 недели)

Цель спринта: перейти от сырого прототипа к стабильному playable build с базовой архитектурой и улучшенным game-feel.

> Старые задачи удалены: текущий файл фиксирует новый, актуальный набор задач на Sprint 1.

## Приоритеты
- **P0** — критично для результата спринта.
- **P1** — важно, но допустим перенос в конец спринта.
- **P2** — желательно, выполняется при наличии времени.

## Задачи

| ID | Приоритет | Задача | Оценка | Результат (Definition of Done) |
|---|---|---|---:|---|
| S1-01 | P0 | Рефакторинг структуры проекта на модули (`game/engine.py`, `game/player.py`, `game/level.py`, `game/ui.py`) | 10ч | Игра запускается как раньше, логика вынесена из монолита `main.py`, нет регрессий по управлению и коллизиям |
| S1-02 | P0 | Добавить dev-mode флаг и отключить покадровый debug print в обычном запуске | 2ч | В стандартном режиме консоль не спамится; debug-логи включаются параметром запуска/конфигом |
| S1-03 | P0 | Улучшение управления: coyote time, jump buffer, variable jump height | 8ч | Прыжок ощущается отзывчиво; поведение покрыто ручными тест-кейсами (пограничные прыжки работают) |
| S1-04 | P0 | Экран `Game Over / Victory` + корректный рестарт уровня | 5ч | После завершения уровня/поражения показывается экран результата и кнопка рестарта возвращает в игру без артефактов |
| S1-05 | P1 | Добавить систему уровней из JSON (минимум 3 коротких уровня в спринте) | 8ч | Уровни грузятся из `data/levels/*.json`, можно переключаться между уровнями |
| S1-06 | P1 | Добавить 2 новых типа интерактивных объектов (напр. шипы и движущаяся платформа) | 7ч | Объекты описаны в уровне, отрисовываются и влияют на игрока по правилам |
| S1-07 | P1 | Базовый UI: стартовое меню, пауза, отображение текущего уровня/счёта | 6ч | Есть минимум 3 состояния UI (menu, play, pause), не ломают игровой цикл |
| S1-08 | P2 | Базовый звук: прыжок, подбор монеты, поражение/победа | 4ч | Звуки проигрываются по событиям, есть глобальный mute |
| S1-09 | P2 | Минимальная телеметрия локально (лучший счёт/лучшее время в файл) | 3ч | После перезапуска приложения рекорд сохраняется |

## Зависимости (порядок выполнения)
1. S1-01 → база для последующих задач.
2. S1-02 + S1-03 → стабилизация core-loop.
3. S1-04 → завершённый игровой цикл.
4. S1-05 → контентная масштабируемость.
5. S1-06 + S1-07 → расширение механик и UX.
6. S1-08 + S1-09 → полировка и удержание.

## План по неделям

### Неделя 1
- S1-01, S1-02, S1-03
- Частично S1-04

### Неделя 2
- Завершить S1-04
- S1-05, S1-06, S1-07
- S1-08, S1-09 (если остаётся буфер)

## Риски и меры
- Риск: регрессии после рефакторинга.  
  Мера: делать рефакторинг малыми PR-частями и ручной smoke-test после каждого шага.
- Риск: недооценка времени на систему уровней.  
  Мера: в спринте ограничить объём до 3 уровней и простого JSON-формата.
- Риск: плохой game-feel.  
  Мера: итеративно подбирать параметры движения по чек-листу ощущений.

## Итоговые критерии Sprint 1
- Playable build с меню, минимум 3 уровнями и базовыми новыми механиками.
- Отзывчивое управление и чистый runtime без лишнего debug-шума.
- Возможность быстро продолжить развитие во Sprint 2 без повторного монолитного роста.
