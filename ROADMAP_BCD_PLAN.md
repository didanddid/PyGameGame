# Roadmap: Этапы B, C, D

Документ фиксирует оставшиеся этапы roadmap (B, C, D) в формате конкретных задач, чтобы мы не потеряли договорённости и могли выполнять их итеративно.

> Этот файл — постоянный бэклог для следующих спринтов после Sprint 1.

## Приоритеты
- **P0** — критично для цели этапа.
- **P1** — важно, допускается перенос в следующий спринт.
- **P2** — улучшения/полировка при наличии буфера.

---

## Этап B — «Из прототипа в игру» (ориентир: 2–4 недели)

Цель: добавить структуру прогрессии и контентный минимум, чтобы игра ощущалась как полноценный продукт.

| ID | Приоритет | Задача | Оценка | Definition of Done |
|---|---|---|---:|---|
| B-01 | P0 | Система загрузки уровней из JSON + валидация схемы | 8ч | Уровни читаются из `data/levels/*.json`, ошибки формата валидируются и не ломают запуск |
| B-02 | P0 | Сделать 10 игровых уровней (кривая сложности от простого к среднему) | 16ч | Есть 10 проходимых уровней, проверен порядок сложности и нет "soft-lock" ситуаций |
| B-03 | P0 | Экран выбора уровня + прогресс прохождения | 7ч | Можно выбрать уровень, отображается статус (закрыт/пройден/идеально пройден) |
| B-04 | P0 | Геймлуп завершения: `Victory/Game Over -> Restart/Next Level` | 6ч | После победы/поражения доступен предсказуемый переход без перезапуска приложения |
| B-05 | P1 | Новые сущности: шипы, движущиеся платформы, батут | 12ч | Все 3 типа объектов интегрированы в JSON уровня и работают по правилам |
| B-06 | P1 | Чекпоинты на длинных уровнях | 6ч | После смерти персонаж возрождается с последнего чекпоинта |
| B-07 | P1 | Система оценки уровня (время/монеты/смерти) + 1–3 звезды | 8ч | После финиша рассчитывается рейтинг, данные сохраняются в профиль прогресса |
| B-08 | P2 | Базовый onboarding (подсказки управления в первых уровнях) | 4ч | Первые 1–2 уровня имеют ненавязчивые подсказки, которые можно отключить |
| B-09 | P2 | UX-полировка меню/паузы/настроек | 5ч | Меню консистентно по навигации, есть mute и возврат в главное меню |

### Порядок выполнения B
1. B-01 -> B-02 -> B-03
2. B-04 параллельно с B-03
3. B-05 -> B-06 -> B-07
4. B-08/B-09 как полировка

---

## Этап C — «Удержание и возвратность» (ориентир: 3–6 недель)

Цель: повысить retention и сформировать долгосрочную мотивацию возвращаться в игру.

| ID | Приоритет | Задача | Оценка | Definition of Done |
|---|---|---|---:|---|
| C-01 | P0 | Система daily challenge (уровень дня по seed) | 10ч | Ежедневный челлендж меняется автоматически, для всех игроков seed дня одинаков |
| C-02 | P0 | Ежедневные/недельные миссии | 10ч | Игрок получает задачи, прогресс учитывается, награды начисляются корректно |
| C-03 | P0 | Локальная таблица рекордов (best time/score) | 5ч | Отдельный экран рекордов, данные сохраняются между сессиями |
| C-04 | P1 | Система достижений (10–15 ачивок) | 8ч | Достижения открываются по условиям, статусы сохраняются |
| C-05 | P1 | Косметические анлоки (скины/трейлы) за прогресс | 9ч | Есть магазин/экран анлоков, визуальные изменения применяются к игроку |
| C-06 | P1 | Серии попыток/стрики и мягкие награды за регулярную игру | 6ч | Стрики считаются корректно, есть UI индикация и выдача наград |
| C-07 | P2 | Подготовка онлайн-лидерборда (API-абстракция + очередь отправки) | 10ч | Реализован слой интеграции, при отсутствии сети данные не теряются |
| C-08 | P2 | LiveOps-конфиг (быстрые правки миссий без кода) | 6ч | Параметры миссий и наград задаются через конфиг-файл |

### Порядок выполнения C
1. C-01 + C-02 (ядро возвратности)
2. C-03 + C-04
3. C-05 + C-06
4. C-07/C-08 как подготовка к масштабированию

---

## Этап D — «Техническая устойчивость и масштаб» (параллельно, по 20–30% времени)

Цель: сделать кодовую базу устойчивой к росту функциональности и контента.

| ID | Приоритет | Задача | Оценка | Definition of Done |
|---|---|---|---:|---|
| D-01 | P0 | Финализировать архитектурные слои (engine/domain/ui/data) и границы зависимостей | 10ч | Модули изолированы, нет циклических импортов, ответственность слоёв формализована |
| D-02 | P0 | Ввести фиксированный timestep/надёжный `dt` для физики | 8ч | Поведение движения стабильно при разном FPS |
| D-03 | P0 | Автотесты на ключевую физику и коллизии | 12ч | Есть набор тестов на прыжок/приземление/столкновения, тесты проходят локально |
| D-04 | P1 | Regression checklist + smoke-test скрипт для каждого релиза | 6ч | Перед релизом запускается единый чек-лист и скрипт smoke-проверок |
| D-05 | P1 | Производительность: профилирование и бюджет кадра на слабом ПК | 10ч | Установлен целевой FPS и измерены узкие места, есть список оптимизаций |
| D-06 | P1 | Логирование и обработка ошибок (dev/prod режимы) | 7ч | Ошибки логируются структурированно, debug-вывод отключён в production |
| D-07 | P2 | CI пайплайн (lint + tests + build) | 8ч | При каждом push запускаются проверки и сборка артефакта |
| D-08 | P2 | Релизный процесс (версионирование, changelog, сборка) | 6ч | Есть documented release flow и воспроизводимая сборка |

### Порядок выполнения D
1. D-01 -> D-02 -> D-03
2. D-04 + D-06
3. D-05
4. D-07 + D-08
